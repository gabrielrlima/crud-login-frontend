# Backlog — índice

Uma lista viva de SDDs por status, sem ciclo de tempo fixo e sem retrospectiva.

## O que entra aqui

- Um único arquivo, `backlog.md`, na raiz do projeto que usar este modelo — não um arquivo por ciclo.
- Quatro seções fixas, mesmos nomes do campo **Status** do SDD (`spec-template.md`): **A fazer**, **Em andamento**, **Em revisão**, **Concluído**.

## O que NÃO entra aqui

- Objetivo de ciclo, data de início/fim, retrospectiva → esse vocabulário não existe neste modelo.
- Detalhe de comportamento de cada item → isso é `specs/SDD-ID-nome.md`, o backlog só lista título e status.

## Como usar

1. Copie [`backlog-template.md`](./backlog-template.md) **uma vez** para `backlog.md` na raiz do projeto.
2. Ao criar um SDD novo, adicione uma linha em "A fazer".
3. Mova a linha entre seções conforme o status muda — não crie um arquivo novo por período de tempo.
4. Antes de mover um SDD para "Concluído", confira o checklist de fechamento no próprio `backlog-template.md`.
